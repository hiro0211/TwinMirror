# TwinMirror フリーミアム戦略

> 親2人の写真から、未来の子どもの顔を生成する iOS アプリのマネタイズ設計ドキュメント。
> TikTok でのバイラル獲得を前提に、無料/課金の切り分けを定義する。

---

## 1. 戦略サマリー

- **バイラル流入を殺さない設計**: オンボーディングで先制ペイウォールを出さず、結果を見せた後で課金トリガーを当てる（Soft then Hard）
- **継続課金の動機を作る**: 「年齢進行」で派手な初回体験、「履歴」で継続利用価値を積み上げ
- **コスト効率重視**: AI モデルは Nano Banana 2 を全ユーザーに統一し、差別化は解像度と機能で行う
- **価格は週課金主軸**: Halo AI 戦略を踏襲（$8.99/週相当の ¥1,200/週）

---

| Fast モード | 1日2回（現状3回→2回に絞る） |
| 生成画像 | 1枚のみ / 標準画質（1K） |
| 年齢選択 | 5歳・10歳の2択のみ |
| ウォーターマーク | TwinMirror ロゴ入り（TikTok でのバイラル装置） |
| 履歴 | **直近2件まで**閲覧可能（3件目を生成すると最古が消える） |
| 保存 | カメラロール保存可（標準解像度） |
| 広告 | リワード広告で +1 生成（任意視聴） |

## 4. 課金メリット優先順位

訴求の強さ順。マーケ・ペイウォール文言・オンボーディングで強調すべき順序。

| 順位 | 特典 | 訴求軸 |
|---|---|---|
| 🥇 1 | 年齢進行 0→20歳の全段階 | ストーリー性・新規体験 |
| 🥈 2 | **無制限の履歴・クラウド保存** ⭐️ NEW | 継続利用・データ資産 |
| 🥉 3 | 3パターン同時生成 | バリエーション・TikTok 拡散誘発 |
| 4 | ウォーターマーク除去 | シェア欲求 |
| 5 | 4K HD 画質 | 印刷・待ち受けニーズ |
| 6 | 生成回数無制限 | ヘビーユーザー |

**設計意図**: 1位（年齢進行）は派手な"単発体験"で初回課金トリガーに、2位（履歴）は"継続的な価値"で解約抑制に。両者で「初回課金 × 継続課金」の役割分担を担う。

## 5. 履歴機能の設計 ⭐️ NEW

### 5.1 体験フロー

```
[生成完了画面]
   ↓
通常通り結果を表示（警告等は出さない）
   ↓ カメラロール保存ボタンは常に表示
   ↓
[Home → 履歴タブ] をタップ
   ↓
無料: 直近2件のみ表示。3件目を見ようとすると「すべての履歴を見るにはプレミアム」ペイウォール
有料: 過去すべての生成結果を閲覧可能（カップル別アルバム・比較ビュー対応）
```

**無料版の履歴ロジック**: アプリ内ストレージに最新2件のみ保持。3件目の生成が完了したタイミングで、最も古い1件を自動削除する FIFO 方式。これにより「3件目を見たければ課金」という自然な誘導が成立する。

### 5.2 なぜ刺さるか（3つの理由）

1. **複利的に価値が増える＝解約しにくい**
   回数制限は「使い切ったら課金」の単発動機。履歴は使えば使うほど資産が積み上がる → 解約 = 資産消失なのでチャーン抑制効果が強い。

2. **TwinMirror のユースケースと噛み合う**
   - 「彼女と試した」「友達カップルと試した」「親の写真で試した」と複数の組合せで遊ぶ
   - 「5歳・10歳・15歳と年齢違いで見たい」と同じカップルでも複数試行
   - → 振り返り・比較ニーズが極めて強い

3. **データロックイン**
   クラウド保存 → 機種変更後も残る → ユーザーは課金継続の心理的圧力を感じる（Apple/Google フォト方式）

### 5.3 先行事例

| サービス | 設計 | 学び |
|---|---|---|
| **Snapchat Memories**（2025年9月導入） | 5GB 無料、超過分 $1.99/月〜$15.99/月 | ⚠️ **後付け導入で大バックラッシュ**。TwinMirror は最初からこの設計で出すべき |
| **Zima: Offline AI** | 無料は直近3件のみ / $9.99 買い切りで全履歴 | AI 生成アプリで「履歴 = 課金特典」の成功例。最初から設計済みのためバックラッシュなし。TwinMirror は直近2件に絞る |
| **Adobe Firefly** | クラウド履歴は Adobe アカウント（有料）紐付け | エンタープライズ標準パターン |

### 5.4 設計上の注意点

| リスク | 対策 |
|---|---|
| Snapchat 型バックラッシュ | **最初からこの設計で出す**。リリース後の後付け導入は NG |
| ユーザーが画像をロスト | カメラロール保存ボタンを目立つ位置に常時配置。警告ダイアログは出さず、自然なUXで保存を促す |
| App Store レビューで否認 | **カメラロール保存は無料でも可能にする**（ユーザーの所有権は守る） |
| プライバシー懸念 | クラウド保存はプレミアムユーザーのみ。同意取得を明確化 |
| データ削除権（GDPR等） | プレミアムユーザーがいつでも履歴削除できる導線を必須 |

### 5.5 ストレージ選択肢

| 選択肢 | メリット | デメリット |
|---|---|---|
| **Cloudflare R2** | 既存 Worker と統合しやすい / 安価 | 自前運用 |
| Firebase Storage | Analytics と統合済み | コスト高 |
| iCloud (CloudKit) | プライバシー◎ / 無料枠あり | iOS 限定・Android 展開時に再設計 |

→ **R2 推奨**（バックエンドは既に Cloudflare Worker）

## 6. AI モデル選択の意思決定

### 結論：全ユーザー Nano Banana 2 固定。Pro は採用しない。

### 6.1 比較表

| 項目 | Nano Banana 2 (Flash) | Nano Banana Pro |
|---|---|---|
| 品質 | Pro 比 **約95%相当** | フル |
| 速度 | **4-8秒** | 10-20秒（2-3倍遅い） |
| 標準解像度コスト | **$0.08/枚** | $0.15/枚（約2倍） |
| 4K 解像度コスト | **$0.16/枚** | $0.30/枚 |
| 解像度対応 | 512 / 1K / 2K / **4K** | 1K / 2K / **4K** |

### 6.2 なぜ Pro を採用しないか

1. **顔生成では NB2 が同等以上**（複数レビュー実証済み）
   - Pro は "磨きすぎ"る傾向があり、リアルな子どもの顔生成には逆効果
   - 顔の一貫性テストでも NB2 が安定（10/10 成功）

2. **コスト 2 倍** → プレミアム原価が膨らみマージン圧迫

3. **速度 2-3 倍遅い** → UX 悪化（TikTok 流入の若年層はせっかち）

4. **5% の品質差は素人ユーザーには知覚不能**

5. **「最上位モデル」はマーケ訴求として地味** — 「年齢進行」「履歴」の方が圧倒的に伝わる

### 6.3 差別化は解像度で

```
無料:       Nano Banana 2 / 1K 解像度  ($0.08/枚)
プレミアム: Nano Banana 2 / 4K 解像度  ($0.16/枚)
```

訴求文言: "印刷可・待ち受け最適・SNS 高画質"

### 6.4 将来オプション（Pro 必須機能）

Yearly / Lifetime プラン向けに将来追加候補：

- **家族集合写真**: 祖父母・兄弟姉妹を含む（14枚参照画像必要 → Pro 必須）
- **シーン合成**: 子どもが学校に通う姿、卒業式など（複雑構成 → Pro 有利）

### 7.1 なぜ週課金主軸か

- エンタメ系 AI アプリは利用が散発的 → 週課金が刺さる
- 心理的障壁が低い（¥1,000 台）
- 即座のキャッシュフロー確保 → クリエイター施策・広告への再投資が可能
- Halo AI の事例（週 $8.99 で 16.5% CVR 達成）を踏襲

## 8. ペイウォール戦略

### 8.1 "Soft then Hard" フロー

```
[起動] → [写真2枚選択] → [年齢選択]
   ↓
[Generate ボタン押下]  ← ★ ここで Hard Paywall（無料枠尽きたら）
   ↓
無料枠残ってれば通過 / 尽きたらペイウォール
   ↓
[Result 表示] → [年齢スライダー操作試行] ← ★ Soft Paywall（プレミアム機能誘導）
   ↓
[TikTok シェアボタン押下] → ウォーターマーク除去オプション表示
   ↓
[Home → 履歴タブ タップ] → ★ Hard Paywall（履歴 = プレミアム限定）
```

### 8.2 重要原則

- オンボーディングで先制ペイウォールは**出さない**（バイラル流入を殺す）
- 「**結果を見せた後**」「**シェア直前**」「**履歴を見ようとした瞬間**」が最強の課金タイミング
- 多様なペイウォール設計: TikTok 流入と Google 検索流入で異なるデザインを使う
- 社会的証明: 「本日◯人が生成」のリアルタイムカウンター表示
- 離脱時オファー: ペイウォール閉鎖後に限定時間オファー（初週 ¥280 など）を提示
- キャンセル時の救済フロー: トライアル解約時に割引やプラン変更を提案
